• git config --global user.name blabla налаштування інформації про користувача, для локальних репозиторіїв
  • git config --global user.email blabal@bla
  • git init створити новий локальний репозиторій
  • git status перераховує нові та змінені файли
  • git add індексує файли для коміта
  • git commit фіксує проіндексовані файли
  • git log дивитись історію змін в репозиторії
  • git checkout дозволяє пересуватись по комітах та гілках
  • git branch створення нової гілки
  • git push завантажити всі зміни з локального на віддалений репозиторій
  • git pull завантажити весь проект що знаходиться в репозиторії, оновлює версію на локальному репозиторії
  • git merge можливість вносити зміни з гілки віддаленого в локальний репозиторій
  • git clone скачати репозиторій разом з історією

P.S.: .gitignore файл дає можливість Git ігнорувати перераховані файли в ньому.
  • git revert відмінити коміти
  • git reset відмінити дію, коміт чи індексацію
  • git rebase склеїти зміни в одну гілку
  • git cherry-pick зміни з одного коміта примінить до іншого
  • git stash тимчасово зберігає незбережені файли
